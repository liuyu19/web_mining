职位描述：

职责描述：
1）对标Google GMS服务运维，负责华为终端云服务及HMS生态服务所用运维平台的长期规划，把握业界devops运维领域的方向和趋势，具有较强的运维平台产品规划能力，确保产品运维能力业界领先。
2）对标Google TOP服务：支付、地图、视频、IOT等，有对应行业TOP应用运维专家经验，熟悉行业最新技术动向，了解关键技术架构以及全球布局，负责对应HMS服务的运维规划，确保行业领先。
3）深入研究运维相关技术，设计、规划以及完善现有运维业务体系及平台建设，支撑团队在资源管理、多活部署、AIOPS监控监控、弹性伸缩、故障分析处理等运维领域能力提升。
4）深度参与业务系统的设计与实施，主导系统架构的可运维性设计、高可用设计；持续优化服务架构、性能。
5）探索、研究新的运维技术方向，并推动在产品和运维侧落地。

关键知识:
1）熟悉运维最新技术动向，并在行业内有一定影响。熟悉业界领先的运维架构，在服务的快速上线、自动化和智能化运维的未来运维体系和技术有深入研究；
2）在BAT、小米、高德等行业知名互联网公司工作10年以上，作为团队Leader或核心成员参与过大型互联网公司运维体系的构建或运维平台的规划和落地。

关键技能:
1）有TB级数据平台运维的设计或规划技能。
2）至少熟悉Java/C/C++/Python等编程技术中的一种，熟悉DevOps。
3）具备优秀的逻辑思维能力、沟通能力，有良好的团队合作精神。

工作经验:
1）BAT、小米、高德等行业知名互联网公司工作10年以上，作为团队Leader或核心成员参与过大型互联网公司运维体系的构建或运维平台的规划和落地。
2）有互联网监控系统、CMDB、运维系统开发设计和维护经验，对运维自动化、智能化转型有深刻的认识和实际操作的经验者优先；
